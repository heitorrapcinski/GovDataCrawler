# Requirements Document

## Introduction

OpenClaws is an autonomous AI personal assistant that orchestrates the analysis of government contract artifacts downloaded by the GovDataCrawler into the `target` folder. OpenClaws runs as a containerized service that leverages IBM Granite LLM (running in a separate, fully isolated container with no internet access) to provide intelligent analysis of contract metadata and PDF attachments. The assistant enables users to ask questions about the collected artifacts and receive answers grounded in Brazilian public procurement legislation.

## Glossary

- **OpenClaws_Agent**: The autonomous AI personal assistant container that orchestrates artifact analysis, manages user interactions, and delegates inference requests to the Granite_Service
- **Granite_Service**: The IBM Granite LLM inference container that processes natural language requests from the OpenClaws_Agent with no internet access
- **Target_Folder**: The `target` directory containing all artifacts downloaded by the GovDataCrawler, organized as `target/contratos.comprasnet.gov.br/{orgao}/{unidade_gestora}/{contract_id}/`
- **Contract_Artifact**: A collection of files within a single contract folder, consisting of a `metadata.json` file and zero or more PDF attachments
- **Metadata_File**: A JSON file named `metadata.json` within each contract folder containing structured contract data (contract_id, orgao, unidade_gestora, contract_number, supplier_name, contract_value, start_date, end_date, object_description, extra_fields, attachments, scraped_at)
- **PDF_Attachment**: A PDF document attached to a contract, stored in the same folder as the corresponding Metadata_File
- **Legislation_Knowledge_Base**: A predefined set of Brazilian public procurement legislation URLs that the OpenClaws_Agent uses as reference context for answering questions
- **User_Query**: A natural language question submitted by the user to the OpenClaws_Agent regarding the analyzed artifacts
- **Analysis_Response**: A natural language answer generated by the OpenClaws_Agent in response to a User_Query, grounded in artifact data and legislation references
- **Container_Orchestration**: The Docker Compose configuration that defines and manages the OpenClaws_Agent and Granite_Service containers together

## Requirements

### Requirement 1: Container Architecture

**User Story:** As a developer, I want OpenClaws and Granite to run in separate containers with strict network isolation, so that the LLM has no internet access while the agent can access both the internet and the target folder.

#### Acceptance Criteria

1. THE Container_Orchestration SHALL define the OpenClaws_Agent as a container with access to the internet and a volume mount to the Target_Folder
2. THE Container_Orchestration SHALL define the Granite_Service as a container with no internet access
3. THE Container_Orchestration SHALL configure a private internal network that allows the OpenClaws_Agent to send inference requests to the Granite_Service on a designated service port
4. THE Granite_Service SHALL accept inference requests exclusively from the OpenClaws_Agent through the private internal network
5. THE Granite_Service SHALL have no network route to any external host outside the private internal network
6. THE OpenClaws_Agent SHALL have read-only access to the Target_Folder volume mount
7. IF the Granite_Service container fails to start, THEN THE Container_Orchestration SHALL exit with a non-zero exit code and log an error message indicating the Granite_Service failed, and SHALL prevent the OpenClaws_Agent from starting
8. IF the Granite_Service becomes unreachable after startup, THEN THE OpenClaws_Agent SHALL stop processing, log an error message indicating loss of connectivity to the Granite_Service, and exit with a non-zero exit code within 30 seconds of detecting the failure

### Requirement 2: IBM Granite LLM Integration

**User Story:** As a developer, I want OpenClaws to use IBM Granite as its language model, so that the assistant leverages a capable open-source LLM for artifact analysis.

#### Acceptance Criteria

1. THE Granite_Service SHALL run the IBM Granite LLM model locally within the container
2. THE Granite_Service SHALL expose an inference API endpoint accessible only on the private internal network
3. WHEN the OpenClaws_Agent sends an inference request, THE Granite_Service SHALL process the request and return a generated text response within 120 seconds
4. THE Granite_Service SHALL support configurable model parameters: temperature (0.0 to 2.0, default 0.7), max tokens (1 to the model maximum output length, default 1024), and top-p (0.0 to 1.0, default 0.9)
5. IF the Granite_Service receives a request that exceeds the model context window, THEN THE Granite_Service SHALL reject the request and return an error response indicating the input exceeds the maximum allowed token count
6. IF the Granite_Service receives a request with a parameter value outside its valid range, THEN THE Granite_Service SHALL reject the request and return an error response indicating which parameter is invalid and its acceptable range
7. IF the Granite_Service is not ready to accept requests because the model is still loading, THEN THE Granite_Service SHALL return an error response indicating the service is unavailable and the request should be retried later

### Requirement 3: Artifact Discovery and Indexing

**User Story:** As a user, I want OpenClaws to automatically discover and index all artifacts in the target folder, so that the assistant can answer questions about any downloaded contract.

#### Acceptance Criteria

1. WHEN the OpenClaws_Agent starts, THE OpenClaws_Agent SHALL scan the Target_Folder recursively to discover all Contract_Artifact folders, where a folder is identified as a Contract_Artifact by the presence of a Metadata_File named `metadata.json`
2. WHEN a Contract_Artifact folder is discovered, THE OpenClaws_Agent SHALL parse the Metadata_File and store all fields defined in the Metadata_File schema (contract_id, orgao, unidade_gestora, contract_number, supplier_name, contract_value, start_date, end_date, object_description, extra_fields, attachments, scraped_at) in an in-memory index
3. WHEN a Contract_Artifact folder contains PDF_Attachment files, THE OpenClaws_Agent SHALL extract text content from each PDF_Attachment up to a maximum of 50 MB per file
4. WHEN the recursive scan of the Target_Folder completes, THE OpenClaws_Agent SHALL log the total number of Contract_Artifact folders discovered and the total number of PDF_Attachment files processed
5. IF a Metadata_File cannot be parsed due to invalid JSON, THEN THE OpenClaws_Agent SHALL log the error with the file path and skip that Contract_Artifact without interrupting the scan of remaining folders
6. IF a PDF_Attachment cannot be read or contains no extractable text, THEN THE OpenClaws_Agent SHALL log a warning with the file path and continue processing the remaining artifacts
7. IF the Target_Folder does not exist or is not accessible at startup, THEN THE OpenClaws_Agent SHALL log an error indicating the path is unavailable and exit with a non-zero status code
8. IF the Target_Folder contains zero Contract_Artifact folders, THEN THE OpenClaws_Agent SHALL log a warning indicating no artifacts were found and continue startup with an empty index

### Requirement 4: Legislation Knowledge Base

**User Story:** As a user, I want OpenClaws to have knowledge of relevant Brazilian procurement legislation, so that the assistant can provide legally-informed answers about contracts.

#### Acceptance Criteria

1. THE OpenClaws_Agent SHALL maintain a Legislation_Knowledge_Base containing the following reference URLs:
   - https://www.planalto.gov.br/ccivil_03/_ato2019-2022/2021/lei/l14133.htm
   - https://www.gov.br/compras/pt-br/nllc
   - https://www.gov.br/compras/pt-br/nllc/lista-de-atos-normativos-e-estagios-de-regulamentacao-da-lei-14133-de-2021.pdf
   - https://www.gov.br/compras/pt-br/nllc/legislacao-14-133-por-tema
   - https://www.planalto.gov.br/ccivil_03/_ato2015-2018/2016/lei/l13303.htm
   - https://www.planalto.gov.br/ccivil_03/leis/l8666cons.htm
   - https://www.planalto.gov.br/ccivil_03/leis/2002/l10520.htm
   - https://www.planalto.gov.br/ccivil_03/_ato2011-2014/2011/lei/l12462.htm
   - https://portal.tcu.gov.br/publicacoes-institucionais/cartilha-manual-ou-tutorial/licitacoes-e-contratos-orientacoes-e-jurisprudencia-do-tcu
   - https://portaldatransparencia.gov.br/entenda-a-gestao-publica/licitacoes-e-contratacoes
2. WHEN the OpenClaws_Agent starts, THE OpenClaws_Agent SHALL fetch and cache the content from each Legislation_Knowledge_Base URL with a maximum timeout of 30 seconds per URL
3. IF a Legislation_Knowledge_Base URL is unreachable or exceeds the 30-second timeout, THEN THE OpenClaws_Agent SHALL log a warning indicating the failed URL and continue startup using any previously cached content for that URL
4. IF the OpenClaws_Agent starts for the first time and no previously cached content exists for one or more Legislation_Knowledge_Base URLs that are unreachable, THEN THE OpenClaws_Agent SHALL log an error indicating the unavailable URLs and start with a degraded Legislation_Knowledge_Base containing only the successfully fetched content
5. WHEN generating an Analysis_Response for a User_Query related to procurement contracts, THE OpenClaws_Agent SHALL include at least one citation identifying the specific law name and article number from the Legislation_Knowledge_Base that supports the response

### Requirement 5: User Query Processing

**User Story:** As a user, I want to ask questions about the downloaded artifacts and receive accurate answers, so that I can understand contract details without manually reading all documents.

#### Acceptance Criteria

1. THE OpenClaws_Agent SHALL provide a command-line interface for the user to submit User_Query messages with a length between 1 and 2000 characters
2. IF a User_Query is empty, blank, or exceeds 2000 characters, THEN THE OpenClaws_Agent SHALL reject the input and display a message indicating the allowed query length
3. WHEN a User_Query is received, THE OpenClaws_Agent SHALL identify Contract_Artifact data where the query terms match fields in the in-memory index (contract_id, contract_number, supplier_name, object_description, or extracted PDF text)
4. WHEN a User_Query is received, THE OpenClaws_Agent SHALL construct a prompt containing the matched artifact data, legislation context from the Legislation_Knowledge_Base related to the query topic, and the User_Query, then send the prompt to the Granite_Service for inference
5. WHILE the Granite_Service is processing an inference request, THE OpenClaws_Agent SHALL display a progress indicator to the user through the command-line interface
6. WHEN the Granite_Service returns a response, THE OpenClaws_Agent SHALL present the Analysis_Response to the user through the command-line interface
7. IF the Granite_Service does not respond within 60 seconds or the connection is refused when a User_Query is submitted, THEN THE OpenClaws_Agent SHALL inform the user that the inference service is temporarily unavailable and suggest retrying
8. IF no Contract_Artifact data matches the User_Query based on the index search, THEN THE OpenClaws_Agent SHALL inform the user that no matching artifacts were found and suggest refining the query

### Requirement 6: Context-Aware Response Generation

**User Story:** As a user, I want OpenClaws to provide responses that reference specific contract data and legislation, so that I can trust the accuracy of the answers.

#### Acceptance Criteria

1. WHEN generating an Analysis_Response, THE OpenClaws_Agent SHALL include references to the specific Contract_Artifact data used to formulate the answer by citing the contract folder path and the relevant metadata fields consulted
2. WHEN generating an Analysis_Response that references content from the Legislation_Knowledge_Base, THE OpenClaws_Agent SHALL cite the specific legislation name and article or section number used as basis for the answer
3. WHEN the Analysis_Response references a specific contract, THE OpenClaws_Agent SHALL include the contract_id and contract_number in the response text adjacent to the referenced claim
4. WHEN a User_Query spans multiple contracts, THE OpenClaws_Agent SHALL aggregate data from all matching Contract_Artifact folders up to a maximum of 20 contracts and present a consolidated Analysis_Response with per-contract references
5. IF the available artifact data is insufficient to fully answer a User_Query, THEN THE OpenClaws_Agent SHALL explicitly label each claim in the Analysis_Response as either "based on available data" or "could not be determined from available artifacts"
6. IF a User_Query spans more than 20 contracts, THEN THE OpenClaws_Agent SHALL inform the user that the response covers the first 20 matching contracts and suggest refining the query to narrow the scope

### Requirement 7: Health Monitoring

**User Story:** As a developer, I want to monitor the health of both containers, so that I can detect and resolve issues quickly.

#### Acceptance Criteria

1. THE Granite_Service SHALL expose a health check endpoint on the private internal network that returns a response indicating whether the service is healthy or unhealthy within 5 seconds
2. THE OpenClaws_Agent SHALL periodically check the Granite_Service health endpoint at a configurable interval between 5 and 300 seconds with a default of 30 seconds
3. WHEN the Granite_Service health check returns a non-successful response or the connection times out after 5 seconds, THE OpenClaws_Agent SHALL log a warning-level message indicating the inference service is unhealthy
4. THE Container_Orchestration SHALL define health check configurations for both the OpenClaws_Agent and the Granite_Service containers specifying the check interval, timeout, number of retries, and start period
5. IF the Granite_Service health check fails 3 consecutive times, THEN THE OpenClaws_Agent SHALL log an error-level message to standard output indicating the inference service is degraded

### Requirement 8: Configuration Management

**User Story:** As a developer, I want to configure OpenClaws behavior through environment variables, so that I can adjust settings without modifying code.

#### Acceptance Criteria

1. THE OpenClaws_Agent SHALL read configuration from environment variables at startup including: Target_Folder path (string, max 4096 characters), Granite_Service endpoint URL (string, max 2048 characters), health check interval (integer, 5–300 seconds), and log level (one of DEBUG, INFO, WARN, ERROR)
2. THE Granite_Service SHALL read configuration from environment variables at startup including: model name (string, max 256 characters), inference port (integer, 1–65535), max context length (integer, 512–131072 tokens), and default generation parameters (temperature as float 0.0–2.0, max tokens as integer 1–8192, top-p as float 0.0–1.0)
3. IF a required environment variable is not set, THEN THE OpenClaws_Agent SHALL use the following default values: health check interval of 30 seconds, log level of INFO, and the Target_Folder path of "./target"
4. IF a required environment variable is not set, THEN THE Granite_Service SHALL use the following default values: inference port of 8080, max context length of 4096 tokens, temperature of 0.7, max tokens of 2048, and top-p of 0.95
5. IF an environment variable contains a value outside its defined valid range or of an incompatible type, THEN THE OpenClaws_Agent SHALL log an error message identifying the variable name, the invalid value received, and the expected format, then exit with a non-zero status code
6. IF an environment variable contains a value outside its defined valid range or of an incompatible type, THEN THE Granite_Service SHALL log an error message identifying the variable name, the invalid value received, and the expected format, then exit with a non-zero status code
7. WHEN the OpenClaws_Agent starts successfully with all configuration validated, THE OpenClaws_Agent SHALL log the active configuration values at INFO level, excluding any sensitive endpoint credentials
